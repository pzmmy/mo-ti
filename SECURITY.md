# 安全策略

感谢您帮助维护 Tolaria 的安全。

如果您认为自己发现了安全漏洞，**请不要在 GitHub 上公开提交 issue**。请改为私下报告。

## 支持的版本

我们目前为以下版本提供安全修复支持：

| 版本 | 支持情况 |
| --- | --- |
| 最新稳定版 | ✅ |
| `main` 分支 | 尽力而为 |
| 旧版本 / 预发布版 | ❌ |

## 报告漏洞

请使用本仓库的 GitHub 私有漏洞报告流程。

请尽可能提供以下信息：

- 问题的简要描述
- 复现步骤或概念验证
- 受影响的版本/提交（如已知）
- 影响评估
- 任何建议的缓解措施

如果问题涉及敏感用户数据、凭据或可实际利用的漏洞，请保持报告私密，不要公开披露细节。

## 预期处理流程

我们将尽力：

- 在几个工作日内确认收到报告
- 复现并评估报告内容
- 如果问题属实，着手修复或制定缓解措施
- 在用户获得合理时间更新后，协调公开披露

## 披露指南

请在公开披露细节之前，给予我们合理的时间进行调查和发布修复。

我们感谢负责任的披露和善意的安全研究。

## 不适用范围的说明

以下情况通常不纳入考虑范围，除非能证明其具有实际安全影响：

- 缺少最佳实践的标头或加固措施，但无实际可利用的漏洞
- 需要用户做出不切实际操作的自 XSS 或编辑器行为
- 仅影响不受支持的旧版本构建的报告
- 纯理论性问题，无合理攻击路径

如果您不确定某问题是否属于安全范畴，仍请私下报告。
