# 拼音模糊搜索

墨屉支持通过拼音对中文笔记进行模糊搜索，让你无需切换输入法即可快速找到目标笔记。

## 工作原理

当你输入的查询包含拉丁字母（a-z）时，墨屉会自动将笔记的标题和内容中的中文字符转换为拼音（无声调），然后进行子串匹配。

例如，搜索 `beijing` 会匹配：

- **北京** → `beijing`
- **北京大学** → `beijingdaxue`
- **北京旅游攻略** → `beijinglvyougonglve`

## CJK 二元组分词

对于包含中文的查询，墨屉还会使用二元组分词（bigram）技术进行部分匹配。这允许你只输入部分关键词就能找到相关笔记。

例如，搜索 `北京` 可以匹配 **北京大学**（共享二元组 "北京"）。搜索 `京大` 也可以匹配 **北京大学**（共享二元组 "京大"）。

## 单一中文字符回退

当你只输入一个中文字符作为查询时（如 `京`），墨屉会回退到简单的字符包含匹配，确保任何包含该字符的笔记都能被找到。

## 使用示例

| 输入查询 | 可匹配的笔记标题 |
|---------|----------------|
| `beijing` | 北京、北京大学、北京旅游攻略 |
| `daxue` | 大学、大学生活、北京大学 |
| `北京` | 北京、北京大学（二元组匹配） |
| `京大` | 北京大学（二元组 "京大"） |
| `笔记` | 笔记、笔记技巧、阅读笔记 |
| `bij` | 笔记、笔记技巧（拼音前缀匹配） |

## 已知限制

### 多音字

墨屉使用的 `pinyin` crate 对于多音字始终返回最常见的读音，不会根据上下文选择正确的读音。这意味着：

- ✅ **行路** → `xinglu`，搜索 `xing` 可以匹配
- ❌ **银行** → `yinhang`，但 crate 返回 `yinxing`，搜索 `hang` **无法** 匹配
- ✅ **行动** → `xingdong`，搜索 `xing` 可以匹配

这是底层拼音库的已知限制。未来版本可能会引入更智能的多音字消歧。

### 非 CJK 文件跳过

对于不包含中日韩（CJK）字符的纯英文文件，墨屉会跳过拼音转换以避免不必要的性能开销。
