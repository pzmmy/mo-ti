# 考研/考公如何用墨屉做笔记

## 场景描述

考研、考公备考需要处理大量的教材、网课和习题资料，知识点分散在各处，难以形成体系。传统笔记软件要么功能过于复杂，要么数据不归自己掌控。墨屉的本地优先 + Markdown 原生特性，让你可以完全掌控备考笔记，配合类型系统与关系链接，将零散的知识点编织成一张可检索的知识网络。

**适用人群**：考研学生、公务员考试备考者、CPA/法考等长期备考学习者。

## 操作指南

1. **创建知识库** — 在墨屉中新建一个名为「考研备考」或「公考复习」的知识库（Vault），作为所有资料的统一容器。
2. **配置类型** — 按照「科目/知识点/真题/错题」等分类创建类型文档，给每篇笔记打上类型标签。参见[创建类型指南](/guides/create-types)。
3. **使用收件箱快速捕获** — 将课堂走神时闪过的疑问、刷题时遇到的盲点，用 `Ctrl+N` 快速捕获到收件箱，之后统一整理。参见[收件箱](/concepts/inbox)。
4. **用 Wikilink 串联知识点** — 在笔记中使用 `[[知识点名称]]` 建立关系链接，让「政治-马原-辩证法」形成可追溯的知识链路。参见[使用 Wikilink](/guides/use-wikilinks)。
5. **定期整理收件箱** — 每周花 10 分钟将收件箱笔记分类、加入类型和关系，保持知识库的整洁。参见[整理收件箱](/guides/organize-inbox)。
6. **利用 Git 做版本备份** — 每次复习迭代后提交一次 Git 快照，随时回溯修改记录，再也不怕误删或改错。参见[连接 Git 远程](/guides/connect-a-git-remote)。
7. **构建自定义视图** — 按科目筛选出所有相关笔记，自定义视图让你复习时只看到当前需要的内容。参见[构建自定义视图](/guides/build-custom-views)。
